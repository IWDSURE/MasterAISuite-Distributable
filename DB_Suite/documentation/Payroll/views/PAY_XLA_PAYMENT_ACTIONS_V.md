# PAY_XLA_PAYMENT_ACTIONS_V

## Details

**Schema:** FUSION

**Object owner:** PAY

**Object type:** VIEW

**Source:** [https://docs.oracle.com/en/cloud/saas/human-resources/oedmh/payxlapaymentactionsv-3692.html](https://docs.oracle.com/en/cloud/saas/human-resources/oedmh/payxlapaymentactionsv-3692.html)

## Columns

- EVENT_ID
- PAYROLL_REL_ACTION_ID
- COST_ACTION_ID
- COST_ACTION_TYPE
- BASE_PROCESS_EFFECTIVE_DATE
- ACCOUNTING_DATE
- PERSON_NAME
- CONVERSION_RATE
- CONVERSION_RATE_TYPE
- TRANSACTION_DATE
- PAYROLL_ID
- REVERSAL_FLAG

## Query

```sql
SELECT PXE.EVENT_ID, TGL_RA.PAYROLL_REL_ACTION_ID, COST_RA.PAYROLL_REL_ACTION_ID COST_ACTION_ID, COST_PA.ACTION_TYPE COST_ACTION_TYPE, COST_PA.EFFECTIVE_DATE BASE_PROCESS_EFFECTIVE_DATE, PXE.ACCOUNTING_DATE, PAY_SLA_PKG.GET_PERSON_NAME(PER.FULL_NAME) PERSON_NAME, null CONVERSION_RATE, PAY_SLA_PKG.GET_CONVERSION_TYPE(COST_PA.LEGISLATIVE_DATA_GROUP_ID, COST_PA.EFFECTIVE_DATE) CONVERSION_RATE_TYPE, to_char(PXE.ACCOUNTING_DATE, 'YYYY/MM/DD') TRANSACTION_DATE, PXE.PAYROLL_ID, PXE.REVERSAL_FLAG FROM PAY_XLA_EVENTS PXE, PAY_PAYROLL_REL_ACTIONS TGL_RA, PAY_ACTION_INTERLOCKS COST_LOCK, PAY_PAYROLL_REL_ACTIONS COST_RA, PAY_PAYROLL_ACTIONS COST_PA, PAY_PAY_RELATIONSHIPS_DN PPR, PER_PERSON_NAMES_F_V PER WHERE TGL_RA.PAYROLL_REL_ACTION_ID = PXE.PAYROLL_REL_ACTION_ID AND COST_LOCK.LOCKING_ACTION_ID = TGL_RA.PAYROLL_REL_ACTION_ID AND COST_RA.PAYROLL_REL_ACTION_ID = COST_LOCK.LOCKED_ACTION_ID AND COST_PA.PAYROLL_ACTION_ID = COST_RA.PAYROLL_ACTION_ID AND COST_PA.ACTION_TYPE = 'CP' AND PPR.PAYROLL_RELATIONSHIP_ID = TGL_RA.PAYROLL_RELATIONSHIP_ID AND PER.PERSON_ID (+) = PPR.PERSON_ID AND COST_PA.EFFECTIVE_DATE BETWEEN PER.EFFECTIVE_START_DATE (+) AND PER.EFFECTIVE_END_DATE (+)
```
