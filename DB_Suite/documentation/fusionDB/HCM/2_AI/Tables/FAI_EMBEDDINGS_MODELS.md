# FAI_EMBEDDINGS_MODELS

This table stores metadata of the embeddings models supported by Oracle.

## Details

**Schema:** FUSION

**Object owner:** FAI

**Object type:** TABLE

**Tablespace:** REFERENCE

**Source:** [https://docs.oracle.com/en/cloud/saas/human-resources/oedmh/faiembeddingsmodels-3765.html#faiembeddingsmodels-3765](https://docs.oracle.com/en/cloud/saas/human-resources/oedmh/faiembeddingsmodels-3765.html#faiembeddingsmodels-3765)

## Primary Key

| Name | Columns |
|------|----------|
| FAI_EMBEDDINGS_MODELS_PK | MODEL_ID |

## Columns

| Name | Datatype | Length | Precision | Not-null | Comments |
|---|---|---|---|---|---|
| MODEL_ID | NUMBER |  | 18 | Yes | Autogenerated number and primary key for this table. |
| MODEL_NAME | VARCHAR2 | 100 |  | Yes | Name of the database embeddings model used to convert unstructured data into numeric values for AI. |
| MODEL_VERSION | VARCHAR2 | 100 |  | Yes | Version number of the embeddings model. |
| ONNX_FILE_NAME | VARCHAR2 | 100 |  | Yes | Name of the ONNX file, an open format for machine learning models, enabling the interchange of models. |
| MODEL_ENDPOINT_REFERENCE | VARCHAR2 | 500 |  |  | For external models, the short name reference to a topology manager record to access the endpoint of the model. |
| STATUS | VARCHAR2 | 40 |  | Yes | Status of the embedding model indicating whether the embedding model has been loaded to the database or whether embeddings have been generated. |
| MODEL_TYPE | VARCHAR2 | 40 |  | Yes | Type of the model, DB_RESIDENT or EXTERNAL, indicating where the model is stored. |
| MODEL_CATEGORY | VARCHAR2 | 40 |  | Yes | Category that indicates the type of data supported by the embeddings model. |
| MODEL_LOADING_DATE | TIMESTAMP |  |  |  | Date the embedding model was loaded into the database. |
| MODEL_LIVE_DATE | TIMESTAMP |  |  |  | Date the embedding model status changed to LIVE. |
| MODEL_DEPRECATED_DATE | TIMESTAMP |  |  |  | Date the embedding model status was changed to DEPRECATED. |
| MODEL_INTRODUCTION_RELEASE | VARCHAR2 | 40 |  |  | Fusion Application release when the model was introduced. |
| CREATED_BY | VARCHAR2 | 64 |  | Yes | Who column: indicates the user who created the row. |
| CREATION_DATE | TIMESTAMP |  |  | Yes | Who column: indicates the date and time of the creation of the row. |
| LAST_UPDATED_BY | VARCHAR2 | 64 |  | Yes | Who column: indicates the user who last updated the row. |
| LAST_UPDATE_DATE | TIMESTAMP |  |  | Yes | Who column: indicates the date and time of the last update of the row. |
| LAST_UPDATE_LOGIN | VARCHAR2 | 32 |  |  | Who column: indicates the session login associated to the user who last updated the row. |
| OBJECT_VERSION_NUMBER | NUMBER |  | 9 | Yes | Used to implement optimistic locking. This number is incremented every time that the row is updated. The number is compared at the start and end of a transaction to detect whether another session has updated the row since it was queried. |
| SEED_DATA_SOURCE | VARCHAR2 | 512 |  |  | Source of seed data record. A value of 'BULK_SEED_DATA_SCRIPT' indicates that record was bulk loaded. Otherwise, specifies the name of the seed data file. |
| ORA_SEED_SET1 | VARCHAR2 | 1 |  | Yes | Oracle internal use only. Indicates the edition-based redefinition (EBR) context of the row for SET1. Context values are Y or N. |
| ORA_SEED_SET2 | VARCHAR2 | 1 |  | Yes | Oracle internal use only. Indicates the edition-based redefinition (EBR) context of the row for SET2. Context values are Y or N. |
