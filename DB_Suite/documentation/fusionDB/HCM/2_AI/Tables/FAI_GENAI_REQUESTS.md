# FAI_GENAI_REQUESTS

This table stores request and response data for generative AI related requests.

## Details

**Schema:** FUSION

**Object owner:** FAI

**Object type:** TABLE

**Tablespace:** Default

**Source:** [https://docs.oracle.com/en/cloud/saas/human-resources/oedmh/faigenairequests-28359.html#faigenairequests-28359](https://docs.oracle.com/en/cloud/saas/human-resources/oedmh/faigenairequests-28359.html#faigenairequests-28359)

## Primary Key

| Name | Columns |
|------|----------|
| FAI_GENAI_REQUESTS_PK | GENAI_REQUEST_ID |

## Columns

| Name | Datatype | Length | Precision | Not-null | Comments |
|---|---|---|---|---|---|
| GENAI_REQUEST_ID | NUMBER |  | 18 | Yes | Autogenerated number and primary key for this table. |
| ORCHESTRATION_FLOW_ID | VARCHAR2 | 80 |  | Yes | Autogenerated alternate key for this table. |
| DESIGN_PATTERN | VARCHAR2 | 80 |  | Yes | The design pattern of the request. For example, a Completion request. |
| DESIGN_CODE | VARCHAR2 | 100 |  | Yes | The design code of the Fusion AI entity associated to the request. For example, the prompt code for a completion request. |
| DESIGN_VERSION | VARCHAR2 | 40 |  | Yes | The design version of the Fusion AI entity associated to the request. For example, the prompt version for a completion request. |
| REQUEST | CLOB |  |  |  | The JSON payload of the generative AI request. |
| RESPONSE | CLOB |  |  |  | The response related to the request. For example, the response returned by the large language model for a completion request. |
| CREATED_BY | VARCHAR2 | 64 |  | Yes | Who column: indicates the user who created the row. |
| CREATION_DATE | TIMESTAMP |  |  | Yes | Who column: indicates the date and time of the creation of the row. |
| LAST_UPDATED_BY | VARCHAR2 | 64 |  | Yes | Who column: indicates the user who last updated the row. |
| LAST_UPDATE_DATE | TIMESTAMP |  |  | Yes | Who column: indicates the date and time of the last update of the row. |
| LAST_UPDATE_LOGIN | VARCHAR2 | 32 |  |  | Who column: indicates the session login associated to the user who last updated the row. |
| OBJECT_VERSION_NUMBER | NUMBER |  | 9 | Yes | Used to implement optimistic locking. This number is incremented every time that the row is updated. The number is compared at the start and end of a transaction to detect whether another session has updated the row since it was queried. |

## Indexes
